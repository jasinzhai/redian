## ❓ 如果还是404，请检查

### 1. 清除浏览器缓存

**Chrome/Edge:**
- 按 `Cmd + Shift + Delete` (Mac) 或 `Ctrl + Shift + Delete` (Windows)
- 勾选"缓存的图片和文件"
- 时间范围选"全部"
- 点击"清除数据"

**Safari:**
- 菜单栏 → Safari → 清除历史记录
- 勾选所有选项
- 点击"清除历史记录"

### 2. 使用无痕/隐私模式打开

- Chrome: `Cmd + Shift + N` (Mac) 或 `Ctrl + Shift + N` (Windows)
- Safari: `Cmd + Shift + N`
- Firefox: `Cmd + Shift + P` (Mac) 或 `Ctrl + Shift + P` (Windows)

在无痕模式中访问网址。

### 3. 检查URL是否正确

确认你访问的是：
```
https://jasinzhai.github.io/wangxiangshanhai-hot-tracker
```

不是：
```
https://jasinzhai.github.io/          ❌ 错误
https://github.com/jasinzhai/...        ❌ 错误
http://...                            ❌ 必须是https
```

### 4. 等待几分钟

有时候GitHub Pages需要几分钟才能完全部署：
- 通常1-2分钟
- 最长可能需要10分钟

### 5. 检查仓库是否为公开（Public）

1. 访问：https://github.com/jasinzhai/wangxiangshanhai-hot-tracker
2. 点击 **Settings**
3. 检查 **Danger Zone** 中的 "Change visibility"
4. 确保仓库是 **Public** 而不是 Private

**如果仓库是Private**：
- 点击 "Change visibility"
- 选择 "Make public"
- 确认更改

## 🛠️ 常见问题

### 问题1：显示"404 File not found"

**原因**：
- URL路径错误
- 仓库是私有的
- index.html不存在

**解决**：
- 检查URL是否正确
- 将仓库改为公开
- 确认有index.html文件

### 问题2：显示"404 This is not the web page you are looking for"

**原因**：
- URL路径错误（可能多了或少了一级目录）

**解决**：
- 确认URL格式：`https://用户名.github.io/仓库名/`
- 不是：`https://用户名.github.io/`

### 问题3：页面空白

**原因**：
- JavaScript加载失败
- hot_data.json无法加载
- 浏览器不兼容

**解决**：
- 打开浏览器开发者工具（F12）
- 查看Console标签页的错误信息
- 确认hot_data.json文件存在
